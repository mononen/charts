# Migrate Existing Helm Chart to Common Library

Use this skill when migrating existing Helm charts to use the common-library-chart.

## High-Level Workflow

### 1. Add Library Dependency

Edit `Chart.yaml`:

```yaml
dependencies:
  - name: mononen-library-chart
    version: "1.0.0"
    repository: "https://mononen.github.io/charts/"
```

### 2. Restructure values.yaml

Transform from flat structure to component-based:

**Before:**
```yaml
replicaCount: 3
image:
  repository: myapp
  tag: latest
service:
  port: 80
```

**After:**
```yaml
global:
  clientCode: myorg
  project: myapp
  env: dev
  namespace: myorg
  imageDefaults:
    pullSecrets:
      - name: regcred

components:
  myapp:
    enabled: true
    containers:
      - name: myapp
        image:
          repository: myapp
          tag: ""  # Set via --set
        ports:
          - name: http
            containerPort: 8080
    service:
      port: 80
```

See `references/VALUES_PATTERNS.md` for complete templates.

### 3. Create Entry Point Template

Create `templates/resources.yaml`:
```yaml
{{- include "common.all" . }}
```

### 4. Update Template References

| Old Reference | New Reference |
|---------------|---------------|
| `.Values.config.clientCode` | `.Values.global.clientCode` |
| `.Values.config.env` | `.Values.global.env` |
| `.Values.networking.domain` | `.Values.global.domain` |
| `.Values.myapp.*` | `.Values.components.myapp.*` |

### 5. Migrate Ingress Configuration

If the existing chart has custom ingress templates with annotations, move the annotations into values:

```yaml
components:
  myapp:
    ingress:
      className: nginx
      annotations:
        # Move existing annotations here
        nginx.ingress.kubernetes.io/rewrite-target: /
      internal:
        enabled: true
      external:
        enabled: false
```

### 6. Delete Redundant Templates

Remove templates now generated by library:
```bash
rm templates/<component>/deployment.yaml
rm templates/<component>/service.yaml
rm templates/<component>/hpa.yaml
rm templates/<component>/pdb.yaml
rm templates/<component>/serviceaccount.yaml
```

### 7. Update Dependencies and Test

```bash
helm dependency update .
helm lint .
helm template test-release . > rendered.yaml
kubectl apply --dry-run=client -f rendered.yaml
```

## Key Changes

| Area | Change |
|------|--------|
| Value paths | Flat to `global.*` and `components.*` |
| Ingress | Custom templates to library-generated with `className` + `annotations` |
| Resource generation | Controlled by `generate.*` flags |
| Primary component | Add `primaryComponent: true` to avoid double suffix |

## Common Issues

See `references/TROUBLESHOOTING_CHARTS.md`:

- **Template not found** - Helper still referenced from deleted file
- **Nil pointer** - Add `| default dict` for nested values
- **Resources not generating** - Check `enabled: true`
- **Double component suffix** - Add `primaryComponent: true`
- **Wrong ingress hosts** - Check `global.env` in env configs

## Migration Checklist

**Pre-Migration:**
- [ ] Understand current template structure
- [ ] Document existing resource generation
- [ ] List custom templates to keep

**Migration:**
- [ ] Add library dependency
- [ ] Restructure values.yaml
- [ ] Create resources.yaml entry point
- [ ] Update template value references
- [ ] Migrate ingress annotations to values
- [ ] Delete redundant templates

**Validation:**
- [ ] `helm dependency update` successful
- [ ] `helm lint` passes
- [ ] `helm template` generates expected resources
- [ ] All legacy value keys removed
- [ ] Environment files use correct paths

## Environment File Updates

Update environment configs to use new paths:

```yaml
# config/prd.yaml
global:
  env: prd

components:
  myapp:
    autoscaling:
      minReplicas: 3
      maxReplicas: 20
    ingress:
      external:
        enabled: true
```
